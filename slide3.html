Acceptance Testing
Unit Testing

Verifies individual functions, methods, or classes in isolation. Tests the smallest testable parts of an application to ensure they work as expected.

Key frameworks: pytest, JUnit, NUnit, Jest

Integration Testing

Verifies that different modules or services work well together. Tests the interaction between integrated units, components, or systems.

Approaches: Top-down, Bottom-up, Big Bang

System Testing

Validates the complete, integrated system against specified requirements. Tests the application's behavior as a whole.

Types: Functional, Performance, Usability, Security

Acceptance Testing

Confirms the system meets business and user requirements. Final verification before deployment to ensure system readiness.

Common methods: User Acceptance Testing (UAT), BDD with Cucumber
