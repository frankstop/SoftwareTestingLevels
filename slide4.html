Unit Testing - Definition & Goals

Unit Testing is the process of testing individual components or functions of software in isolation from the rest of the application to validate that each unit performs as expected.

Key Goals of Unit Testing

Early Bug Detection

Identify and fix issues at the earliest stage of development, significantly reducing costs and effort.

Drive Software Design

Test-Driven Development (TDD) uses unit tests to guide implementation and ensure modular, testable code.

Living Documentation

Tests serve as executable documentation, showing how code is intended to be used.

Quick Feedback Loop

Unit tests should run fast (< 50ms each) to provide immediate feedback to developers during development.

Safety Net for Changes

Provides confidence when refactoring code or implementing new features, ensuring existing functionality remains intact.
