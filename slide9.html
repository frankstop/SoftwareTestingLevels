Integration Testing

Integration testing verifies that different modules or services used by your application work together as expected. It focuses on the interactions between components rather than the components themselves.

Top-Down Testing

Starts with high-level modules, gradually working down to lower-level components.

  • Uses stubs to replace lower-level modules
  • Early UI/UX validation
  • Harder to isolate failures at lower levels
Bottom-Up Testing

Begins with lower-level modules, gradually integrating higher-level components.

  • Uses drivers to invoke lower modules
  • Easier to isolate failures
  • UI validation comes later in the process
Big Bang Testing

Combines all modules at once for integration testing.

  • Simple approach for smaller systems
  • Difficult to isolate errors
  • Later integration can lead to discovered issues
Sandwich/Hybrid Testing

Combines both top-down and bottom-up approaches.

  • More comprehensive coverage
  • Requires more resources
  • Best for complex systems with many dependencies

Narrow vs. Broad Integration Tests

Modern approaches favor narrow integration tests that focus only on the code that interacts with external services, using test doubles for those services.
