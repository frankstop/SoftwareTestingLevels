Unit Testing Best Practices

Test One Thing Per Test

Each test should verify a single behavior or concept. This makes tests more maintainable, easier to understand, and provides clearer failure messages.

Good: test_add_user_with_valid_email()

Use Descriptive Test Names

Name tests to clearly express what functionality they're verifying and what conditions they're testing under.

Avoid: test_user()
Better: test_user_registration_fails_with_existing_email()

Use Mocks Sparingly

Mock external dependencies only, not internal implementation details. Overuse of mocks leads to brittle tests that break when implementation changes.

When to mock: APIs, databases, file systems, external services

Keep Tests Fast

Unit tests should execute quickly (< 50ms each). Slow tests discourage frequent test runs and slow down the development feedback loop.

Tip: Profile slow tests to identify bottlenecks and consider moving them to a separate test suite if they can't be optimized.

The Arrange-Act-Assert Pattern

1

Arrange

Set up test preconditions

2

Act

Execute system under test

3

Assert

Verify expected outcomes

# Example following AAA pattern
def test_calculate_total():
# Arrange
cart = ShoppingCart()
cart.add_item(Item("book", 10.0))
cart.add_item(Item("pen", 5.0))
# Act
total = cart.calculate_total()
# Assert
assert total == 15.0
